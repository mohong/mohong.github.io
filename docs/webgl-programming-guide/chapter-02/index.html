#最短的WebGL程序：清空绘图区

function main() {
    const canvas = document.getElementById('webgl')
    const gl = getWebGLContext(canvas)
    // 指定绘图区域的背景颜色
    gl.clearColor(0.0, 0.0, 0.0, 1.0)
    // 将指定缓冲区设定为预定的值，如果清空的是颜色缓冲区，那么将使用gl.clearColor()指定的值作为预设值。
    gl.clear(gl.COLOR_BUFFER_BIT)
  }

gl.clearColor(red, green, blue, alpha)

指定绘图区域的背景色：所有参赛的值均为[0.0, 1.0]。

清空绘图区域，实际上是在清空颜色缓冲区（color buffer）。

gl.clear(buffer)

将指定缓冲区设定为预定的值。如果清空的是颜色缓冲区，那么将使用gl.clearColor()指定的值（作为预定值）。

参数：

名称说明可选值
buffer指定待清空的缓冲区，位操作符OR(|)可用来指定多个缓冲区gl.COLOR_BUFFER_BIT：指定颜色缓存；gl.DEPTH_BUFFER_BIT：指定深度缓冲区；gl.STENCIL_BUFFER_BIT：指定模板缓冲区；

返回值：无

#绘制一个点（版本1）

// 顶点着色器
const VSHADER_SOURCE = 
      'void main() {\n' +
      ' gl_Position = vec4(0.5, 0.5, 0.0, 1.0);\n' + // 设置坐标
      ' gl_PointSize = 10.0;\n' + // 设置尺寸
      '}\n';

// 片源着色器
const FSHADER_SOUCE = 
      'void main() {\n' +
      ' gl_FragColor = vec4(0.0, 1.0, 0.0, 1.0);\n' + // 设置颜色
      '}\n';

function main() {
  const canvas = document.getElementById('webgl')
  const gl = getWebGLContext(canvas)

  // 初始化着色器
  if (!initShaders(gl, VSHADER_SOURCE, FSHADER_SOUCE)) {
    return
  }

  // 指定绘图区域的背景颜色
  gl.clearColor(0.0, 0.0, 0.0, 1.0)
  // 将指定缓冲区设定为预定的值，如果清空的是颜色缓冲区，那么将使用gl.clearColor()指定的值作为预设值。
  gl.clear(gl.COLOR_BUFFER_BIT)

  // 绘制一个点
  gl.drawArrays(gl.POINTS, 0, 1);
}

#着色器是什么

webgl需要两种着色器：

  • 顶点着色器（Vertex shader）：顶点着色器是用来描述顶点特性（如位置、颜色等）的程序。顶点（vertex）是指二维或三维空间中的一个点。
  • 片元着色器（Fragment shader）：进行逐片元处理过程如光照的程序。片元（fragment）是一个webgl术语，你可以将其理解为像素（图像的单元）。

从执行JavaScript程序到在浏览器中显示结果的过程：

GLSE中的数据类型

类型描述
float表示浮点数
vec4表示由四个浮点数组成的矢量，(float, float, float, float)

顶点着色器的内置变量

类型和变量名描述
vec4 gl_Position表示顶点位置，必须被赋值，否则着色器无法正常工作
vec4 gl_PointSize表示点的尺寸（像素数）

片元着色器的内置变量

类型和变量名描述
vec4 gl_FragColor指定片元颜色（RGBA格式）

齐次坐标

由4个分量做成的矢量被称为齐次坐标。

gl.drawArrays(mode, first, count)

执行顶点着色器，按照mode参数指定的方式绘制图形。

参数：

名称类型说明可选值
mode指定绘制的方式gl.POINTS，gl.LINES，gl.LINE_STRIP，gl.LINE_LOOP，gl.TRIANGLES，gl.TRIANGLE_STRIP，gl.TRIANGLE_FAN
first整型指定从哪个顶点开始绘制
count整型指定绘制需要用到多少个顶点

#WebGL坐标系统

通常，在WebGL中，当你面向计算机屏幕时，X轴是水平的（正方形为右），Y轴是垂直的（正方向为下），而Z轴垂直于屏幕（正方向为外）。符合右手坐标系。

webgl坐标与<canvas>坐标的对应关系如下：

  • canvas的中心点：(0.0, 0.0, 0.0)
  • canvas的上边缘和下边缘：（-1.0, 0.0, 0.0）和（1.0, 0.0, 0.0）
  • canvas的左边缘和右边缘：（0.0, -1.0, 0.0）和（0.0, 1.0, 0.0）

#绘制一个点（版本2）

这个版本实现了从JavaScript中传输顶点的位置坐标到顶点着色器中：

// 顶点着色器
const VSHADER_SOURCE = 
      'attribute vec4 a_Position;\n' +
      'attribute float a_PointSize;\n' +
      'void main() {\n' +
      ' gl_Position = a_Position;\n' + // 设置坐标
      ' gl_PointSize = a_PointSize;\n' + // 设置尺寸
      '}\n';

// 片源着色器
const FSHADER_SOUCE = 
      'void main() {\n' +
      ' gl_FragColor = vec4(0.0, 1.0, 0.0, 1.0);\n' + // 设置颜色
      '}\n';

function main() {
  const canvas = document.getElementById('webgl')
  const gl = getWebGLContext(canvas)

  // 初始化着色器
  if (!initShaders(gl, VSHADER_SOURCE, FSHADER_SOUCE)) {
    return
  }

  // 获取attribute变量的存储位置
  const a_Position = gl.getAttribLocation(gl.program, 'a_Position');
  const a_PointSize = gl.getAttribLocation(gl.program, 'a_PointSize');
  // 将顶点位置传输给attribute变量
  gl.vertexAttrib3f(a_Position, 0.5, 0.0, 0.0);
  gl.vertexAttrib1f(a_PointSize, 25.0);

  // 指定绘图区域的背景颜色
  gl.clearColor(0.0, 0.0, 0.0, 1.0)
  // 将指定缓冲区设定为预定的值，如果清空的是颜色缓冲区，那么将使用gl.clearColor()指定的值作为预设值。
  gl.clear(gl.COLOR_BUFFER_BIT)

  // 绘制一个点
  gl.drawArrays(gl.POINTS, 0, 1);
}

gl.getAttribLocation(program, name)

获取由name参数指定的attribute变量的存储地址。

参数：

名称说明
program指定包含顶点着色器和片元着色器的着色器程序对象
name指定想要获取其存储地址的attribute变量的名称

返回值：

返回值说明
大于等于0attribute变量的存储地址
-1指定的attribute变量不存在，或者其命名具有gl_或者webgl_前缀

gl.vertexAttrib3f(location, v0, v1, v2)

将数据（v0, v1, v2）传给由location参赛指定的attribute变量。

参数：

名称说明
location指定将要修改的attribute变量的存储位置，即gl.getAttribLocation返回的值
v0指定填充attribute变量的第一个分量的值，浮点型
v1指定填充attribute变量的第二个分量的值，浮点型
v2指定填充attribute变量的第三个分量的值，浮点型

返回值：无

同族函数：

  • gl.vertexAttrib1f(location, v0)
  • gl.vertexAttrib2f(location, v0, v1)
  • gl.vertexAttrib3f(location, v0, v1, v2)
  • gl.vertexAttrib4f(location, v0, v1, v2, v3)

该系列函数的任务就是从JavaScript向顶点着色器中的attribute变量传值。未设置的值，第四个会被填充为1.0，其他的会被填充为0.0。

也可以使用这些方法的矢量版本，它们的名字以“v”（vertor）结尾，并接受类型化数组作为参数，函数名中的数字表示数组中的元素个数。比如：

const position = new Float32Array([1.0, 2.0, 3.0, 1.0]);
gl.vertexAttrib4fv(a_Position, position)

#通过鼠标点击绘点

客户端坐标转换为webgl坐标

浏览器客户区中的坐标与canva中的坐标

canvas的坐标系统和webgl中的坐标系统

// 鼠标点击位置的坐标
let x = ev.clientX;
let y = ev.clientY;

const rect = ev.target.getBoundingClientRect();

x = (x - rect.left) - canvas.width / 2) / (canvas.width / 2);
y = (canvas.height / 2 - (y - rect.top)) / (canvas.height / 2);
  • rect.left和rect.top是canvas的原点在浏览器客户区中的坐标；
  • 这样（x-rect.left）和（x-rect.top）就可以将客户区坐标系下的坐标（x，y）转换伟canvas坐标系下的坐标了
  • 接下来将canvas坐标系下的坐标转换到webgl坐标系中，通过（canvas.height /2）和（canvas.width/2）求出canvas的中心点
  • 使用（x - rect.left) - canvas.width / 2）和（canvas.height / 2 - (y - rect.top)）将canva的圆点平移到中心点（webgl坐标系统的原点位于此处）
  • 因为webgl中轴的坐标区间为[-1, 1]，所以最后一步需要将x坐标除以（canvas.width/2），y坐标除以（canvas.height/2）

#改变点的颜色

只有顶点着色器才能使用attribute变量，使用片元着色器时，就需要使用uniform变量。或者varying变量。

uniform变量用来从JavaScript程序向顶点着色器和片元着色器传输“一致的”（不变的）数据。

uniform变量的声明

在GLSL ES中，你只能指定float类型的attribute变量，但是却可以指定任意类型的uniform变量。

gl.getUniformLocation(program, name)

获取指定名称的uniform变量的存储位置

参数：

名称说明
program指定包含顶点着色器和片元着色器的着色器程序对象
name指定想要获取其存储位置的uniform变量的名称

返回值：

返回值说明
non-null指定uniform变量的位置
null指定的uniform变量不存在，或者其命名具有gl_或者webgl_前缀

gl.uniform4f(location, v0, v1, v2, v3)

将数据（v0, v1, v2, v3）传输给由location参数指定的uniform变量。

参数：

名称说明
location指定将要修改的uniform变量的存储位置
v0指定填充uniform变量的第一个分量的值
v1指定填充uniform变量的第二个分量的值
v2指定填充uniform变量的第三个分量的值
v3指定填充uniform变量的第四个分量的值

返回值：无

同族函数：

gl.uniform1f(location, v0)

gl.uniform2f(location, v0, v1)

gl.uniform3f(location, v0, v1, v2)

gl.uniform4f(location, v0, v1, v2, v3)
